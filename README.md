# Music Streaming Service (CLI)

## 1. Project Overview

This project is a command-line music streaming simulator written in Python.
It loads songs from a JSON file, lets users search and browse songs, simulates song streaming, and allows speed control and user rating.

## 2. Key Features

- Main menu driven CLI interface.
- Search modes:
  - Search by pattern: `<song> by <artist>`
  - Search by title
  - Search by artist
- Browse all songs and select one to play.
- Speed multiplier with keyboard controls:
  - Left arrow: slower
  - Right arrow: faster
  - `q`: stop playback
- Song rating system (1 to 5 stars).
- Rating data is persisted back to `songs.json`.

## 3. Project Structure

- `main.py`: service entry point and all CLI flow (`StreamService`).
- `Song.py`: `Song` model and JSON serialisation (`to_dict`).
- `Artist.py`: `Artist` model and artist-song association.
- `songs.json`: song dataset (title, duration, genre, rating info, artist).

## 4. Data Model

### Song (`Song.py`)

Fields:

- `title` (str)
- `duration` (int, seconds)
- `genre` (str)
- `rating_num` (int)
- `avg_rating` (float or `"N/A"`)
- `artist` (`Artist` object or `None`)
- `id` (auto-incremented in runtime)

Behaviour:

- `to_dict()` converts in-memory object back to JSON format.
- If artist is missing, `artist` is saved as `null` in JSON.

### Artist (`Artist.py`)

Fields:

- `name` (str)
- private song list (`__songs`)

Behaviour:

- `add_song(song_name)` keeps track of song titles by this artist.

## 5. Input Data Format (`songs.json`)

Each item is a JSON object with the following shape:

```json
{
  "title": "Shape of You",
  "duration": 233,
  "genre": "Pop",
  "rating_num": 10,
  "avg_rating": 4.9,
  "artist": "Ed Sheeran"
}
```

Notes:

- `artist` can be `null`.
- `avg_rating` may be a float greater than 0 and less than or equal to 5, or `"N/A"`.

## 6. Runtime Flow (`main.py`)

1. `StreamService` initialises.
2. Songs are loaded from `songs.json`.
3. Artist objects are reused by name to avoid duplicates.
4. Main menu loops until user quits.
5. While playing, the user can control speed and stop with keyboard input.
6. After playing, the user can submit a rating, which updates the song’s average rating and is then saved to the JSON file, or simply return to the menu.

## 7. How to Run

### Requirements

- Python 3.x
- Dependencies in `requirements.txt` (notably `keyboard`)

### Install

```bash
pip install -r requirements.txt
```

### Start

```bash
python main.py
```

## 8. Current Behaviours and Constraints

- Search is exact-match (case and spacing matter).
- Data persistence currently updates only rating fields back into JSON.
- UI is terminal-only and clears screen between views.
- Keyboard hook based controls may require OS permissions on some environments.

## 9. Possible Improvements

- Add stronger error handling for file I/O and invalid selections.
- Use specific exception types instead of broad `except` blocks.
- Add fuzzy/case-insensitive search.
- Add unit tests for parsing, search, and rating logic.
- Split service logic and UI rendering into separate modules.

## 10. Author Notes

This project demonstrates OOP basics (`Song`, `Artist`, and service orchestration), JSON-based persistence, and interactive CLI control flow.
The data in songs.json was partially created by the author but was mostly generated by AI (ChatGPT and GitHub Copilot).
